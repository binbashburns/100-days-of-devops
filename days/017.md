# Day 017: Install and Configure PostgreSQL

**Time**: 10 minutes

## Objective
The `Nautilus` application development team has shared that they are planning to deploy one newly developed application on `Nautilus` infra in `Stratos DC`. The application uses PostgreSQL database, so as a pre-requisite we need to set up PostgreSQL database server as per requirements shared below:
(PostgreSQL database server is already installed on the Nautilus database server.)
- Create a database user `kodekloud_rin` and set its password to `LQfKeWWxWD`.
- Create a database `kodekloud_db8` and grant full permissions to user `kodekloud_rin` on this database.

## Technologies Used
- `psql`
- PostgreSQL Commands

## Steps

1. SSH into `stdb01` server

    ```bash
    thor@jumphost ~$ ssh peter@stdb01
    [peter@stdb01 ~]$
    ```

2. Create a database user `kodekloud_rin` and set its password to `LQfKeWWxWD`

    ```bash
    # Postgres shell
    [peter@stdb01 ~]$ sudo -u postgres psql

    # Create user
    # https://www.postgresql.org/docs/current/sql-createuser.html
    postgres=# CREATE USER kodekloud_rin WITH ENCRYPTED PASSWORD 'LQfKeWWxWD';
    ```

3. Create a database `kodekloud_db8` and grant full permissions to user `kodekloud_rin` on this database

    ```bash
    # Create database
    # https://www.postgresql.org/docs/current/sql-createdatabase.html
    postgres=# CREATE DATABASE kodekloud_db8;

    # Permissions
    # https://www.postgresql.org/docs/18/sql-grant.html
    postgres=# GRANT ALL PRIVILEGES ON DATABASE kodekloud_db8 TO kodekloud_rin;

    # Exit
    postgres=# \q
    ```

## Key Takeaways
- Read developer docs!
