# Day 028: Git Cherry Pick

**Time**: 15 minutes

## Objective
The Nautilus application development team has been working on a project repository `/opt/blog.git`. This repo is cloned at `/usr/src/kodekloudrepos` on `storage server` in `Stratos DC`. They recently shared the following requirements with the DevOps team:

- Identify the two branches: `master` and `feature`.
- Find the commit on `feature` with the message `Update info.txt`.
- Cherry-pick that commit onto `master`.
- Push the updated `master` branch.

## Technologies Used
- `git`
- `ssh`

## Steps

1. SSH into `ststor01` server, navigate to `/usr/src/kodekloudrepos/blog`, checkout `feature`

    ```bash
    # SSH to Storage Server
    thor@jumphost ~$ ssh natasha@ststor01

    [natasha@ststor01 ~]$

    # Change directories
    [natasha@ststor01 ~]$ cd /usr/src/kodekloudrepos/blog/
    
    # Observe branches in the repository
    [natasha@ststor01 blog]$ sudo git branch

    # * feature
    #   master

    # Looks like we are already on the correct branch
    ```

2. Find the commit referenced in the prompt, and cherry-pick it, and commit/push it to master

    ```bash
    # Check git log
    [natasha@ststor01 blog]$ sudo git log

    # commit 4a81b00de89f4df9803baab01ff8c4977de38892 (HEAD -> feature, origin/feature)
    # Author: Admin <admin@kodekloud.com>
    # Date:   Wed Dec 17 11:24:20 2025 +0000

    #     Update welcome.txt

    # commit 8257709e2dfba17773aea45bf0ffe18025352551
    # Author: Admin <admin@kodekloud.com>
    # Date:   Wed Dec 17 11:24:20 2025 +0000

    #     Update info.txt

    # commit 102fce84a62e13c882927d5f45f017a8ec58844b (origin/master, master)
    # Author: Admin <admin@kodekloud.com>
    # Date:   Wed Dec 17 11:24:20 2025 +0000

    #     Add welcome.txt

    # commit b353e766d2dddd52a21cb95a8bf3840aa7962d02
    # Author: Admin <admin@kodekloud.com>
    # Date:   Wed Dec 17 11:24:20 2025 +0000

    #     initial commit

    
    # The commit we are after is 8257709e2df[...]

    # Switch to master
    [natasha@ststor01 blog]$ sudo git switch master

    # Cherry pick commit
    [natasha@ststor01 blog]$ sudo git cherry-pick 8257709e2df

    # [master 83fa012] Update info.txt
    # Date: Wed Dec 17 11:24:20 2025 +0000
    # 1 file changed, 1 insertion(+), 1 deletion(-)

    # push changes
    [natasha@ststor01 blog]$ sudo git push
    ```

## Key Takeaways
- Cherry-picking can be relatively straightforward, it's just important to remember the steps
